## Goal

在当前工作区从 0 到 1 落地一个**比赛可演示版**“翼修通”MVP，完成以下单一目标：

- 打通“故障输入 -> 知识检索 -> 排故推理 -> 工单生成 -> 人工确认 -> 反馈留痕”的完整闭环

明确不做：

- 不接入真实企业系统
- 不做真实设备控制
- 不做复杂权限体系
- 不做多租户、部署集群、运维平台
- 不做大规模模型训练

---

## Scope & Assumptions

### IN Scope

- Web 演示端 1 套
- FastAPI 后端 1 套
- 维修/工艺/案例模拟知识库 1 套
- RAG 检索链路
- 多 Agent 编排链路
- 规则约束与风险提示
- 工单草案生成
- 人工确认与反馈回流
- 离线评测脚本与比赛 Demo 数据包

### OUT of Scope

- 企业 ERP/MES/EAM/PLM 接口
- 实时视频流分析
- 复杂 OCR 生产级优化
- 真正的航空适航审计
- 自研大模型训练或微调平台
- 原生移动端

### Assumptions

- 技术栈固定为 `FastAPI + LangGraph + React(Vite) + SQLite/JSON + FAISS`
- 主 LLM 优先使用 [Qwen/Qwen2.5-7B-Instruct](https://hf.co/Qwen/Qwen2.5-7B-Instruct)
- 向量模型优先使用 [BAAI/bge-m3](https://hf.co/BAAI/bge-m3)
- 重排模型优先使用 [BAAI/bge-reranker-v2-m3](https://hf.co/BAAI/bge-reranker-v2-m3)
- Agent 编排参考 [langchain-ai/langgraph](https://github.com/langchain-ai/langgraph)
- 首版只支持“文本故障描述 + 故障码输入”；图片理解作为后续增强项
- 首版数据为模拟航空制造/运维知识包，不直接依赖真实企业数据
- 团队至少有 3 个角色：后端/Agent、数据/RAG、前端/演示

## Milestones

### M1: 工程骨架与模拟数据准备

目标：

- 项目能启动
- 模拟数据可读
- 领域对象模型确定

完成标志：

- 后端 health route 可用
- 前端空壳页可打开
- 样例数据集可被程序加载

### M2: 检索与规则能力落地

目标：

- 完成索引构建
- 完成召回与重排
- 完成基础规则约束

完成标志：

- 输入 `E-204` 能召回相关手册和历史案例
- 规则库能命中高风险条件并给出提醒

### M3: Agent 闭环与 API 集成

目标：

- 串起任务路由、排故推理、工单生成、审计拦截
- 对外暴露工作流 API

完成标志：

- 一次请求可返回诊断结果、证据片段、工单草案、风险提示

### M4: 演示界面、评测和比赛封装

目标：

- 前端可完整走通 Demo
- 离线评测可输出指标
- Demo 数据包与初始化脚本可直接运行

完成标志：

## Risks & Mitigations

### 风险 1：没有真实航空数据，导致专业感不足

- Mitigation:
  - 使用结构化模拟知识包，不用零散示例文本
  - 每条答案必须返回来源片段
  - 明确系统验证的是“闭环能力”，不是企业生产部署

### 风险 2：模型出现幻觉，给出无依据建议

- Mitigation:
  - 先检索再回答
  - 审计 Agent 检查证据数量和规则命中状态
  - 无依据时只输出“需人工确认/证据不足”

### 风险 3：多 Agent 编排过重，影响交付速度

- Mitigation:
  - 只做 4 个核心 Agent：router、diagnosis、work_order、audit
  - 质检协同、图像理解全部后置

### 风险 4：前端过度设计导致延期

- Mitigation:
  - 演示端只保留一个主页面
  - 不做登录、权限、复杂导航

### 风险 5：本地模型推理速度太慢

- Mitigation:
  - 首选云 API
  - 本地仅保留可替换接口
  - 预热索引和固定 Demo 样例

### 风险 6：评测来不及做

- Mitigation:
  - T014 只做最小评测集
  - 仅统计 3 个指标：证据命中率、步骤完整率、风险提示命中率

---

## Test Strategy

- 每个任务只新增 1 个最小验证测试
- 优先验证主链路，而不是覆盖率
- 测试分三类：
  - 单元测试：schema、rules、retrieval、agents
  - API 测试：工作流接口
  - 脚本测试：索引构建、评测、Demo 启动

主链路最终验收只看 1 个场景：

- 输入 `E-204 + 振动异常 + 温度偏高`
- 输出：
  - 相关证据片段
  - Top3 可能原因
  - 检查步骤
  - 风险提示
  - 工单草案
  - 人工确认入口
